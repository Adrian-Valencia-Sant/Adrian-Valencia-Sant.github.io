Python Game: Blackjack

The development of our Python Game, Blackjack, was created through repl.it which allowed Sarah Olson and I to test our code live and supports numerous languages.

We had difficulties when it came to calling global variables in our functions which would be called through a while loop to always run our game.

We solved this issue by setting the state to global in order to be accessible by other variables. Consequently, we set states in every function to allow the while loop to run through and execute any code necessary and this resolved our issues and provided the functionality of our game. This was a collaborative project, and I had one partner, Sarah Olson.
